# SWM Wiki

Living knowledge base that feeds agents, cron jobs, tools, and product decisions.

## Structure

```
wiki/
  knowledge/    # Foundational knowledge articles (cron, infra, patterns)
  research/     # Research outputs from scheduled agents (market, competitive, technical)
  agents/       # Agent definitions and configurations
  playbooks/    # Operational playbooks (deployment, testing, product launches)
```

## How it works

1. **Knowledge** is written by humans or compiled from research
2. **Agents** read knowledge to become more effective
3. **Cron jobs** run agents on schedules, pointed at research directions
4. **Research outputs** flow back into the wiki
5. **Better knowledge** produces better agents, better cron jobs, better research

This is a compounding flywheel. Every cycle makes the next one sharper.
